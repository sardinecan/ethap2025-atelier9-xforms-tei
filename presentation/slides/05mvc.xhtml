L'architecture MVC de XForms

Un document XForms est un document XHTML avec :

  • un modèle (<xf:model/>), placé dans l'en-tête HTML (<head/>), contenant les instances de données (<xf:instance/>) ;
  • La vue régit l'affichage et la présentation des données. Elle détermine comment les données du modèle sont représentées à l'écran. La vue est placée dans le corps du document (<body/>)
  • La couche contrôleur gère les interactions entre la vue et le modèle. Elle orchestre la mise à jour des données en réponse aux actions des utilisateurs et peut exécuter des actions spécifiques comme la soumission des données.

L'ensemble doit être servi comme un document XML (extension .xml ou .xhtml).
